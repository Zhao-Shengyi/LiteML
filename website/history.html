开发历程 / Development History

LiteML 编译器的完整开发记录 — 关键决策、Bug 修复和经验教训。

Complete development log — key decisions, bug fixes, and lessons learned.

2026-07-22
🎨 CSS 模板系统 / Template System

创建 12 款 CSS 模板，支持 @template(name) 一行引用：

  • 模板涵盖极简、现代、暗色、玻璃态、新拟态、复古等风格
  • 每款模板包含 theme.css + 中英双语 example.lite/example.html
  • CLI 新增 templates list 子命令
  • 每个模板使用 CSS 变量，用户可覆盖自定义
2026-07-22
🔧 语法清理 + Bug 修复 / Syntax Cleanup

注释前缀从 @ 改为 !，组件引用从 @use 改为 use。旧语法向后兼容。

修复 .class 和 #id 简写问题 — 省略标签名时默认 div。

2026-07-21
🏗️ 重构：模块拆分 + 组件系统 + TUI / Refactoring

将单文件 liteml.py（1046 行）拆分为 core/ 包：

  • constants.py — 常量定义
  • models.py — 数据结构（Node, ParsedLine, ComponentInfo）
  • parser.py — .lite 语法解析、行类型分类、预处理
  • directive_loader.py — @directive 装饰器注册 + 自动发现
  • directive_macros.py — 独立宏 (@audio, @video, @flex...)
  • component.py — 组件加载器
  • compiler.py — 编译核心
  • cli.py — 命令行入口 + TUI

实现了 @use 组件系统，支持约定式和清单式发现。

创建了 Textual TUI（core/tui.py），提供可视化文件浏览 + 编辑 + 编译预览。

2026-07-21
💻 VS Code 扩展 / VS Code Extension

创建完整的 VS Code 扩展，支持：

  • TextMate 语法高亮覆盖全部 token 类型
  • 诊断检查器检测缩进、未知标签、PHP 配对等 9 类错误
  • 一键编译（Ctrl+Alt+B）
  • 自定义编译器路径

修复了 linter 将 JS 检查和编译器检查串联而非短路的问题。

2026-07-21
⚙️ 编译器开发 / Compiler Development

创造了 liteml.py（约 1000 行 Python 标准库），支持：

  • HTML/PHP 双模式输出
  • 缩进嵌套 + CSS 选择器语法
  • 布局指令（@flex, @grid, @waterfall 等）
  • 交互指令（@modal, @theme-toggle, @like）
  • PHP 模板引擎（@if, @foreach, {{ $var }}）
  • Eject 回退机制、@raw 块、长文本、代码块
  • CLI build/watch 命令
2026-07-21
📝 语法文档 + 双语化 / Documentation

创建了 SYNTAX.md — 完整的 LiteML 语法参考手册（1090 行，13 章）。

将 README.md 和 SYNTAX.md 全部中英双语化。

编写了 prompts/（AI 写 LiteML / 反编译 HTML 到 LiteML）。

关键决策 / Key Decisions

开发过程中做的一些重要技术选择：

决策 / Decision 方案 / Choice 理由 / Rationale
编译语言 Python 标准库 零依赖，即开即用
模块结构 core/ 包拆分 单一 liteml.py 达 1000+ 行，难以维护
指令系统 @directive 装饰器 + 插件发现 新增指令只需创建一个 .py 文件
组件加载 约定式 + component.json 简单场景直接放文件，复杂场景显式声明
CSS 模板 @template 宏内联读取 无需额外构建步骤，一行引用
注释语法 ! 前缀 与 @ 指令区分，避免混淆
.class 简写 默认 div 符合直觉，减少重复书写

已知问题 / Known Issues

当前版本的已知限制：

  • Markdown 图片语法 ![alt](src) 在重构中丢失 — 可用标准 img[src=url][alt=text] 替代
  • @audio/@video 宏与标准标签功能重复，未来可能移除
  • attribute 值包含空格时可能出现引号嵌套问题
  • 尚无 CI/CD（GitHub Actions）
  • 尚无自动化测试框架
