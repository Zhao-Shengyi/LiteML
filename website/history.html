🏗️ 开发历程 / Development History

从概念到实现的关键决策、Bug 修复和经验教训。完全开源。

2026-07-22
🎨 官网 + CSS 模板系统 / Website & Theme System

创建官方网站（LiteML 自举构建），同步修复属性值引号嵌套 Bug。

完成 12 款 CSS 模板，@template(name) 一行引用，每款配中英双语示例。

2026-07-22
🔧 语法清理 + .class 简写修复 / Syntax Cleanup

注释从 @ → !（@ 兼容）；组件引用 @use → use。

修复 .container、#main 等简写不被识别的问题：省略标签名时默认 div。

2026-07-21
🏗️ 重构：模块拆分 / Refactoring into core/ Package

单文件 liteml.py（1046 行）→ core/ 包，8 个模块各司其职：

  • parser / compiler / cli / tui 独立
  • directive_loader + directive_macros + directives/ 插件体系
  • component.py 组件加载器

同时创建了 Textual TUI（core/tui.py），实现可视化编辑预览。

2026-07-21
💻 VS Code 扩展 / VS Code Extension

完整的 VS Code 扩展，包含：

  • TextMate 语法高亮覆盖全部 token
  • 诊断检查器（缩进、未知标签、PHP 配对等 9 类）
  • 一键编译（Ctrl+Alt+B）

修复了 linter 中 JS 检查和编译器检查的短路问题。

2026-07-21
⚙️ 编译器开发 / Compiler Development

用 Python 标准库实现完整编译器，约 1000 行。支持：

  • HTML / PHP 双模式输出
  • 缩进嵌套 + CSS 选择器语法
  • 布局 / 交互指令 + LiteJS 无代码交互
  • PHP 模板引擎（@if, @foreach, {{ $var }}）
  • Eject 回退、@raw 块、长文本、代码块
  • CLI build / watch 命令

修复了 6 个核心 Bug（缩进栈、@else 输出、PHP 指令关闭等）。

2026-07-21
📝 语法文档 + 双语化 / Documentation

创建 SYNTAX.md（1090 行，13 章），全部中英双语。

编写 AI 提示词（prompts/）：教 AI 写 LiteML、反编译 HTML 回 LiteML。

关键决策 / Key Decisions

决策 方案 理由
编译语言 Python 标准库 零依赖，即开即用
模块结构 core/ 包拆分 单文件 1000+ 行难以维护
指令系统 @directive 装饰器 + 自动发现 新增指令只需创建一个 .py 文件
组件加载 约定式 + component.json 简单场景直接放文件，复杂场景显式声明
CSS 模板 @template 宏内联读取 无需额外构建步骤
注释语法 ! 前缀 与 @ 指令区分，避免混淆

已知问题 / Known Issues

  • Markdown 图片语法 ![alt](src) 在重构中丢失 — 可用 img[src=url][alt=text] 替代
  • attribute 值含空格时已修复引号嵌套问题
  • 尚无 CI/CD（GitHub Actions）
  • 尚无自动化测试框架
